## 핵심 설정

- 이 학교는 전학생을 대상으로 **기억 조작 실험**을 수행하는 위장 시설
- 전학 후 **5일 내에 '최종 처리'** (기억 완전 소거 또는 영구 격리)가 이루어짐
- 은수(관리), 리인(약물), 세아(감시)가 각자의 역할을 수행
- 설화는 1년 전 실험이 역효과를 일으켜 오히려 모든 기억을 갖게 된 케이스 → '처리' 후 유령으로 학교에 묶임
- 유나는 설화의 사건을 목격하고 카메라로 증거를 모아온 유일한 내부 저항자

## 호감도-패널티 시스템 (Affinity as Penalty)

플레이어가 1~3일차에 쌓은 호감도가 4~5일차에서 탈출 난이도로 전환되는 핵심 시스템.

```
Day 1~3: 선택 → 호감도 ↑ (달콤한 로맨스)
              ↓
Day 3 밤: 장르 전환 — "호감도"의 정체는 "위험도"
              ↓
Day 4~5: 높은 호감도 = 강한 방해 = 어려운 탈출
```

### 설계 원칙
- **호감도 = 족쇄**: 1~3일차의 감정적 투자가 4~5일차의 공포로 전환
- **딜레마**: 캐릭터 개별 스토리를 보려면 호감도를 올려야 하지만, 탈출은 더 어려워짐
- **유나만 예외**: 유나와의 유대만이 탈출에 양의 효과
- **탈출 불가능은 없음**: 호감도 MAX여도 탈출 루트는 존재. 단, 난이도와 긴박감이 상승

### 스탯 구조 (StateManager.js)

| 변수 | Phase 1 표시 | Phase 2 표시 | 범위 | 설명 |
|------|-------------|-------------|------|------|
| `affinity` | "호감도" (위장) | 그대로 | -100~100 | 친밀도 |
| `danger` | 숨김 | "위험도" | -100~100 | 실제 위험 수준 |
| `trust` | 숨김 | "신뢰도" | -100~100 | 유나에게만 의미 |

**위장 공식**: `displayAffinity = affinity × 0.6 + danger × 0.4`
→ danger가 높을수록 "호감도"도 높아 보이는 함정

### 엔진 구현 사양

#### 1. 약물 타이머 감산

`GameEngine._loadScene()` 내에서 `timedChoice` 처리 시:
```js
if (this.state.hasFlag('drank_riin_drink')) {
    timedChoice = Math.max(2000, timedChoice - 2000);
}
```
→ 최소 2초 보장. Day 4 이후 모든 타이머 선택지에 적용.

#### 2. 호감도 분기 (기존 `affinityBranches` 활용)

```js
// 시나리오 파일 예시 — Day 5 세아 조우
"day5_lunch_sea_encounter": {
    affinityChar: "sea",
    affinityBranches: [
        { minAffinity: 60, next: "day5_sea_block_max" },
        { minAffinity: 40, next: "day5_sea_block_high" },
        { minAffinity: 20, next: "day5_sea_block_mid" },
        { minAffinity: 0,  next: "day5_sea_pass" }
    ]
}
```

#### 3. 플래그 기반 분기 (기존 `branches` 활용)

```js
// Day 5 리인 약물 경유
"day5_riin_check": {
    branches: [
        { condition: "drank_riin_drink", next: "day5_riin_drug_detour" }
    ],
    next: "day5_riin_normal"
}
```

#### 4. 과거 선택 콜백 (Day 5 대면 장면)

Day 5 추격 시퀀스에서 세아/리인/은수 조우 시, 플레이어의 과거 선택에 따라 대사가 분기:
```js
// Day 5 세아 조우 — 과거 선택 콜백
"day5_sea_callback": {
    branches: [
        { condition: "day1_choco_accept", next: "day5_sea_line_choco" },
        { condition: "day1_lunch_sea", next: "day5_sea_line_lunch" },
        { condition: "day2_sea_vulnerability", next: "day5_sea_line_promise" }
    ],
    next: "day5_sea_line_default"
}
```
→ 조건이 여러 개 충족되면 **가장 마지막(최근)** 선택을 우선 표시.
→ 플레이어가 직접 한 선택이 그대로 되돌아오므로 심리적 충격 극대화.

#### 5. 시야 흐림 글리치 (약물 후유증)

`drank_riin_drink` 플래그 활성 시, Day 4~5의 타이머 선택지 직전에 0.5초 블랙아웃 + 텍스트 오염:
```js
glitch: {
    drugEffect: true,
    blackoutDuration: 500,
    textCorruption: true,       // Zalgo 노이즈 간헐 삽입
    choiceWave: true,           // 선택지 글씨 흔들림
    choiceDisguise: danger >= 40 // 선택지 위장 (dgr≥40일 때만)
}
```

### 글리치 구현 규칙
- JSON에서 `string` 값(object가 아닌)은 글리치 텍스트로 처리
- `boolean` 값(`true`)은 UI 글리치 트리거로 처리
- 글리치는 0.3~0.5초 동안만 표시 후 정상 복귀
- 플레이어가 "봤나?" 싶은 순간에 사라져야 함
